Guardar tu Trabajo (Commit)

Hacer un commit es como tomarte una selfie cuando estás arreglado. Es una fotografía perfecta del estado de tu código en ese preciso momento, para que nunca pierdas tu avance diario.

Pasos en la consola:

1. Dile a Git que prepare TODOS los archivos que editaste (los pone en el carrito de compras):

<пре>git add .

2. Saca la foto y ponle un título descriptivo (NUNCA pongas "asdasd"):

<пре>git commit -m "Mi primer taller de BD resuelto"

Cambiar de Rama (Branching)

Las ramas (branches) te permiten trabajar en universos paralelos. Puedes intentar cosas raras, dañar el código y romper tu base de datos en una rama nueva, mientras la rama principal (main) sigue hermosa y sin errores.

Crear y Viajar a una nueva rama: <пре>git checkout -b mi-nueva-tarea

Esto crea la rama "mi-nueva-tarea" y te teletransporta a ella de inmediato.

Volver a tu rama principal (viajar entre mundos): <пре>git checkout main

Vuelves sano y salvo a tu código original seguro.

Subir a GitHub (Push)

¡Por fin! Hiciste el taller genial, ya tomaste la foto (commit) y ahora quieres enviarla al profesor o a tus compañeros. Eso se hace subiendo tus avances a la nube con Push.

Manda tus cambios hacia la rama correspondiente en internet:

git push origin main

⚠️ Si no estás en main y estás en "mi-nueva-tarea", obvio debes lanzar: git push origin mi-nueva-tarea
