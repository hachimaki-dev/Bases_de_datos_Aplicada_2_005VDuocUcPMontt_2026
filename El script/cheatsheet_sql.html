Consultas Básicas y Manipulación de Datos

SELECT … FROM … WHERE … ORDER BY

Estructura fundamental de toda consulta. El WHERE filtra filas; el ORDER BY ordena el resultado final.

SELECT col1, col2, col3
FROM nombre_tabla
WHERE condicion
ORDER BY col1 ASC, col2 DESC;
Ejemplo de salida:
COL1 COL2 COL3
A 300 Activo
A 120 Inactivo
B 500 Activo

Alias de Columnas (AS)

Renombra una columna en el resultado. No altera la tabla original.

SELECT nombre AS nombre_empleado,
       sueldo * 1.1 AS sueldo_reajustado
FROM empleado;

Operadores de Comparación y Lógicos

  • =, != o <> — Igualdad / desigualdad.
  • >, <, >=, <= — Comparación numérica o de fechas.
  • BETWEEN a AND b — Rango inclusivo.
  • IN (v1, v2, v3) — Pertenencia a un conjunto.
  • LIKE 'patrón%' — Búsqueda parcial (% = cualquier cantidad de caracteres, _ = uno solo).
  • IS NULL / IS NOT NULL — Verificación de nulos.
  • AND, OR, NOT — Combinación de condiciones.

INSERT INTO … SELECT Clave para Req. 4

Carga una tabla destino con los resultados de una consulta. No requiere PL/SQL. Ideal para poblar tablas históricas.

INSERT INTO tabla_destino (col1, col2, col3)
SELECT c1, SUM(c2), c3
FROM tabla_origen
WHERE condicion
GROUP BY c1, c3;
Atención: Las columnas del SELECT deben coincidir en cantidad, orden y tipo con las del INSERT INTO.

Funciones de Caracteres (Texto)

Operan sobre valores de tipo VARCHAR2 / CHAR y devuelven texto o un número.

UPPER / LOWER / INITCAP

Transforman la capitalización de una cadena completa.

SELECT UPPER('hola mundo')   -- HOLA MUNDO
     , LOWER('HOLA MUNDO')   -- hola mundo
     , INITCAP('hola mundo') -- Hola Mundo
FROM DUAL;

SUBSTR(cadena, inicio, longitud) Clave para Req. 1

Extrae una porción de texto. En Oracle, las posiciones comienzan en 1 (no en 0).

-- Extraer el 3er dígito de un RUN almacenado como número:
SELECT SUBSTR(TO_CHAR(numrun_emp), 3, 1) FROM empleado;

-- Primeras 2 letras del apellido materno en mayúscula:
SELECT UPPER(SUBSTR(apmaterno_emp, 1, 2)) FROM empleado;
Ejemplo con RUN = 15234567:
Expresión Resultado
TO_CHAR(15234567) '15234567'
SUBSTR('15234567', 3, 1) '2'

Concatenación ( || )

Une cadenas de texto. Es el operador estándar de Oracle para concatenar.

-- Construir correo para Req. 1:
SELECT SUBSTR(TO_CHAR(e.numrun_emp), 3, 1)
    || UPPER(SUBSTR(e.apmaterno_emp, 1, 2))
    || '@motorent.cl' AS correo
FROM empleado e;
Resultado para RUN 15234567, apellido materno "González":
CORREO
2GO@motorent.cl

TRIM / LENGTH / REPLACE

  • TRIM(cadena) — Elimina espacios al inicio y final.
  • LENGTH(cadena) — Devuelve la cantidad de caracteres.
  • REPLACE(cadena, 'buscar', 'reemplazo') — Sustituye todas las ocurrencias. 
SELECT TRIM('  Hola  ')             -- 'Hola'
     , LENGTH('Oracle')              -- 6
     , REPLACE('2026-03-31', '-', '/') -- '2026/03/31'
FROM DUAL;

NVL(expr, valor_si_nulo) Muy Útil

Devuelve un valor alternativo cuando la expresión es NULL. Evita espacios vacíos o errores de concatenación.

-- Si el segundo nombre es NULL, usar cadena vacía:
SELECT pnombre || ' ' || NVL(snombre, '') || ' ' || appaterno
FROM empleado;

Funciones de Fecha y Conversión

SYSDATE y Literales de Fecha

SYSDATE retorna la fecha y hora actual del servidor. Para usar una fecha fija, utilice el formato literal ANSI.

SELECT SYSDATE FROM DUAL;                -- Fecha actual
SELECT DATE '2026-03-31' FROM DUAL;      -- Literal ANSI (recomendado)
SELECT TO_DATE('31/03/2026', 'DD/MM/YYYY') FROM DUAL; -- Conversión explícita

EXTRACT(componente FROM fecha) Clave para Req. 4

Extrae una parte numérica de una fecha: YEAR, MONTH o DAY.

-- Filtrar arriendos del año 2026:
SELECT * FROM arriendo_moto
WHERE EXTRACT(YEAR FROM fecha_ini_arriendo) = 2026;

ADD_MONTHS(fecha, n) Clave para Req. 2

Suma (o resta) n meses a una fecha. Útil para calcular antigüedad sin depender de MONTHS_BETWEEN.

-- Empleados contratados DESPUÉS del 31/03/2006 ==> menos de 20 años al 31/03/2026
SELECT * FROM empleado
WHERE fecha_contrato > ADD_MONTHS(DATE '2026-03-31', -240);
¿Por qué -240?
Cálculo Resultado
20 años × 12 meses 240 meses
ADD_MONTHS('2026-03-31', -240) 31/03/2006

MONTHS_BETWEEN(fecha1, fecha2)

Calcula la diferencia en meses entre dos fechas. Devuelve un número decimal.

-- Alternativa: calcular años de servicio directamente
SELECT nombre,
       TRUNC(MONTHS_BETWEEN(DATE '2026-03-31', fecha_contrato) / 12) AS anios_servicio
FROM empleado;

TO_CHAR / TO_DATE / TO_NUMBER

Funciones de conversión explícita entre tipos de datos.

  • TO_CHAR(fecha, 'formato') — Fecha → Texto.
  • TO_CHAR(numero) — Número → Texto (necesario antes de usar SUBSTR sobre un número).
  • TO_DATE('texto', 'formato') — Texto → Fecha.
  • TO_NUMBER('texto') — Texto → Número.
SELECT TO_CHAR(SYSDATE, 'DD "de" MONTH "de" YYYY') FROM DUAL;
-- Resultado: 14 de ABRIL   de 2026

SELECT TO_CHAR(numrun_emp) FROM empleado;
-- Convierte el RUN numérico a texto para poder usar SUBSTR

ROUND / TRUNC (Numéricos)

  • ROUND(numero, decimales) — Redondea al decimal indicado.
  • TRUNC(numero, decimales) — Trunca sin redondear.
SELECT ROUND(45678.567, 0)   -- 45679
     , TRUNC(45678.567, 0)   -- 45678
     , ROUND(45678.567, 2)   -- 45678.57
FROM DUAL;

Funciones de Grupo (Agrupación)

Procesan múltiples filas y devuelven un solo valor por grupo. Se usan con GROUP BY.

Funciones de Agregado Clave para Req. 3

  • COUNT(*) — Cuenta filas del grupo (incluyendo nulos).
  • COUNT(col) — Cuenta filas donde la columna no es NULL.
  • SUM(col) — Suma los valores numéricos.
  • AVG(col) — Promedio aritmético.
  • MIN(col) / MAX(col) — Valor extremo.

GROUP BY — Regla de Oro

Toda columna en el SELECT que no esté dentro de una función de agregado debe aparecer en el GROUP BY. Violar esta regla produce un error ORA-00937.

-- Correcto:
SELECT tipo_moto, COUNT(*) AS cantidad, AVG(valor_arriendo_dia) AS promedio
FROM motocicleta m
JOIN tipo_moto tm ON tm.id_tipo_moto = m.id_tipo_moto
GROUP BY tipo_moto
ORDER BY cantidad DESC;
Resultado esperado:
TIPO_MOTO CANTIDAD PROMEDIO
Deportiva 5 48600
Touring 4 42250
Scooter 3 22333

HAVING — Filtrar Grupos

WHERE filtra filas antes de agrupar. HAVING filtra grupos después de agrupar.

-- Solo grupos con más de 2 registros:
SELECT depto_id, COUNT(*) AS total
FROM empleado
GROUP BY depto_id
HAVING COUNT(*) > 2;
Error frecuente: Usar WHERE COUNT(*) > 2 genera ORA-00934. Las funciones de grupo solo se filtran con HAVING.

Joins — Combinación de Tablas

Permiten relacionar dos o más tablas mediante sus claves primarias (PK) y foráneas (FK).

INNER JOIN (JOIN) Obligatorio para Req. 1, 2, 3, 4

Devuelve solo las filas donde hay coincidencia en ambas tablas. Es el join por defecto y el más utilizado.

SELECT m.placa, m.modelo, tm.nombre_tipo_moto
FROM motocicleta m
JOIN tipo_moto tm
    ON tm.id_tipo_moto = m.id_tipo_moto;
Visualización conceptual:
Tabla A (motocicleta) Tabla B (tipo_moto)
id_tipo_moto = 1 ⟷ id_tipo_moto = 1, "Deportiva"
id_tipo_moto = 2 ⟷ id_tipo_moto = 2, "Scooter"
id_tipo_moto = 9 ✗ (no existe → se descarta)

LEFT JOIN (LEFT OUTER JOIN)

Devuelve todas las filas de la tabla izquierda. Si no hay coincidencia en la derecha, las columnas de esta última aparecen como NULL.

SELECT m.placa, a.fecha_ini_arriendo
FROM motocicleta m
LEFT JOIN arriendo_moto a
    ON a.placa = m.placa;

Útil para detectar motos sin arriendos (las que aparecen con NULL en la columna de arriendo).

Join Múltiple (3+ tablas)

Se encadenan varios JOIN consecutivamente. Cada ON establece una relación distinta.

-- Req. 1 usa 3 tablas: motocicleta + tipo_moto + empleado
SELECT tm.nombre_tipo_moto, m.placa, m.modelo, e.appaterno_emp
FROM motocicleta m
JOIN tipo_moto tm  ON tm.id_tipo_moto = m.id_tipo_moto
JOIN empleado e    ON e.numrun_emp    = m.numrun_emp;

Errores Comunes con Joins

  • ORA-00918: column ambiguously defined — Falta el alias de tabla (m.placa en vez de solo placa) cuando la columna existe en más de una tabla.
  • Producto cartesiano accidental — Olvidar el ON genera un cruce de TODAS las filas con TODAS las filas.
  • Orden de los joins — El motor optimiza internamente, pero la legibilidad mejora si empieza por la tabla principal.

Conocimientos Adicionales

Comandos y patrones avanzados que amplían las capacidades expresivas de SQL. No son estrictamente necesarios para la evaluación, pero fortalecen la formación profesional.

CASE WHEN — Lógica Condicional Bonus

Crea una columna calculada basada en condiciones lógicas, equivalente a un IF/ELSE en programación.

SELECT nombre, sueldo,
    CASE
        WHEN sueldo < 500000  THEN 'Tramo 1'
        WHEN sueldo < 1000000 THEN 'Tramo 2'
        ELSE 'Tramo 3'
    END AS tramo_sueldo
FROM empleado;

Subconsultas en WHERE Bonus

Una consulta dentro de otra. La subconsulta se ejecuta primero y su resultado alimenta la consulta externa.

-- Empleados con sueldo mayor al promedio:
SELECT nombre, sueldo
FROM empleado
WHERE sueldo > (SELECT AVG(sueldo) FROM empleado);

Funciones Analíticas (Window Functions) Bonus

Calculan agregados sin colapsar las filas. Cada fila conserva su detalle individual y obtiene además un valor global o por partición.

SELECT nombre, sueldo, depto_id,
       AVG(sueldo) OVER (PARTITION BY depto_id) AS prom_depto,
       RANK()      OVER (ORDER BY sueldo DESC)  AS ranking
FROM empleado;
Resultado conceptual:
NOMBRE SUELDO DEPTO_ID PROM_DEPTO RANKING
Ana 1200000 10 950000 1
Pedro 700000 10 950000 2
Luis 650000 20 625000 3
Sofía 600000 20 625000 4

Observe que PROM_DEPTO se repite para cada empleado del mismo departamento sin necesidad de GROUP BY.

DUAL — Tabla de Prueba de Oracle

DUAL es una tabla especial de Oracle con una sola fila y una sola columna. Se usa para evaluar expresiones sin necesitar una tabla real.

SELECT SYSDATE FROM DUAL;
SELECT 2 + 2 FROM DUAL;
SELECT SUBSTR('123456', 3, 1) FROM DUAL;  -- Resultado: '3'
