DML y Memoria

Las operaciones INSERT, UPDATE y DELETE no son inmediatas en el disco. Se guardan en el Segmento UNDO hasta que decidas qué hacer con ellas.

COMMIT

El "Guardar Partida". Hace que los cambios transaccionales pendientes sean permanentes y visibles para el resto de los usuarios.

ROLLBACK

El botón de "Pánico". Deshace todas las operaciones de la transacción actual devolviendo la base de datos a su estado anterior.

El Peligro del WHERE

Olvidar el WHERE en un UPDATE o DELETE significa aplicar la operación sobre toda la tabla. ¡Destrucción asegurada!

Viaje en el tiempo: SAVEPOINT

Ya sabes que ROLLBACK deshace toda la transacción. Pero, ¿qué pasa si estás haciendo un trabajo gigante (ej. insertar 50 registros), te equivocas en el número 49, y no quieres perder el trabajo de los 48 anteriores?

Aquí entra SAVEPOINT nombre_punto;. Funciona exactamente igual que los "Checkpoints" en un videojuego.

Creas un punto de guardado temporal dentro de tu transacción. Si cometes un error más adelante, puedes retroceder solamente hasta ese punto usando ROLLBACK TO nombre_punto;, sin perder lo que hiciste antes de él. 

INSERT INTO emp (id, nom) VALUES (1, 'Ana');
INSERT INTO emp (id, nom) VALUES (2, 'Beto');

-- Creamos el punto de guardado
SAVEPOINT paso_seguro;

-- Seguimos trabajando
INSERT INTO emp (id, nom) VALUES (3, 'Carlos');
UPDATE emp SET salario = 9999999; -- ¡Opps! Error catastrófico, actualicé a todos

-- Retrocedemos en el tiempo solo hasta el punto seguro
ROLLBACK TO paso_seguro;

-- Estado final: Ana y Beto existen. Carlos no se insertó y el salario NO se actualizó.
COMMIT;

Subconsultas en DML

Una Subconsulta es simplemente un SELECT metido dentro de otra instrucción (entre paréntesis). El motor primero resuelve el SELECT de adentro, y usa el resultado para completar la instrucción de afuera.

En cláusulas WHERE (Filtros dinámicos)

Imagina que quieres borrar a los empleados que ganan menos que el promedio. ¿Cómo sabes el promedio? ¡Con una subconsulta!

DELETE FROM empleados 
WHERE salario < (SELECT AVG(salario) FROM empleados);

En INSERT (Poblamiento masivo sin VALUES)

A veces necesitas copiar miles de filas de una tabla a otra (ej. para un respaldo). En lugar de escribir miles de VALUES, puedes reemplazar la palabra VALUES entera por un gran SELECT.

Regla de oro: Las columnas del SELECT deben coincidir en cantidad y tipo de dato con las columnas que estás insertando.

-- Copiando todos los empleados del departamento 10 a la tabla de respaldo
INSERT INTO emp_respaldo (id_emp, nombre, depto)
SELECT id, nombre_completo, id_departamento 
FROM empleados 
WHERE id_departamento = 10;

Inserciones Complejas: INSERT ALL

El INSERT normal solo puede enviar datos a una sola tabla. ¿Qué pasa si quieres leer registros y enviarlos a múltiples tablas distintas dependiendo de alguna condición lógica? Para esto existe INSERT ALL.

Inserción Condicional (Cláusula WHEN)

Como si fuera un IF/ELSE, puedes evaluar cada fila leída por el SELECT final, y decidir en qué tabla cae.

-- Leemos la tabla empleados (SELECT final)
-- Si ganan más de un millón van a 'emp_vip', sino a 'emp_normales'.
INSERT ALL
    WHEN sueldo > 1000000 THEN
        INTO emp_vip (id, nombre, sueldo) VALUES (emp_id, emp_nom, sueldo)
    WHEN sueldo <= 1000000 THEN
        INTO emp_normales (id, nombre, sueldo) VALUES (emp_id, emp_nom, sueldo)
SELECT emp_id, emp_nom, sueldo FROM empleados;

Pivoteo en la Misma Tabla

También sirve para insertar varias filas en la misma tabla de un solo golpe. Para lograr esto sin leer datos reales, se usa una tabla especial de Oracle llamada DUALTabla mágica interna de Oracle que tiene exactamente 1 columna y 1 fila. Se usa para operaciones que requieren la cláusula FROM sintácticamente pero no necesitan consultar datos reales (ej. cálculos matemáticos o funciones del sistema)..

-- Insertamos 3 roles nuevos de una sola vez
INSERT ALL
    INTO roles (id_rol, desc) VALUES (1, 'Admin')
    INTO roles (id_rol, desc) VALUES (2, 'Editor')
    INTO roles (id_rol, desc) VALUES (3, 'Lector')
SELECT 1 FROM DUAL; -- El motor ejecuta las inserciones 1 vez (porque DUAL tiene 1 fila)

Demostración en Vivo

script_demo_avanzado.sql 
-- 1. SETUP DE TABLAS
CREATE TABLE temp_origen (id NUMBER, val VARCHAR2(20), sueldo NUMBER);
CREATE TABLE temp_altos (id NUMBER, val VARCHAR2(20));
CREATE TABLE temp_bajos (id NUMBER, val VARCHAR2(20));

-- Poblando origen con INSERT ALL (Pivoteo)
INSERT ALL
    INTO temp_origen VALUES (1, 'Juan', 500)
    INTO temp_origen VALUES (2, 'Ana', 1500)
    INTO temp_origen VALUES (3, 'Beto', 800)
SELECT 1 FROM DUAL;

COMMIT;
SELECT * FROM temp_origen; -- Verificamos que existen

-- 2. DEMO SAVEPOINT
UPDATE temp_origen SET sueldo = sueldo + 100 WHERE id = 1;
SAVEPOINT despues_de_juan; -- Punto de control

DELETE FROM temp_origen; -- Error terrible! Borramos todo sin querer
SELECT * FROM temp_origen; -- Está vacía!! Pánico!

ROLLBACK TO despues_de_juan; -- Retrocedemos al punto de control
SELECT * FROM temp_origen; -- Fiuuu... están de vuelta y Juan tiene su aumento.
COMMIT;

-- 3. DEMO INSERT ALL CONDICIONAL
-- Repartimos los datos a las tablas altos y bajos según el sueldo
INSERT ALL
    WHEN sueldo > 1000 THEN
        INTO temp_altos (id, val) VALUES (id, val)
    WHEN sueldo <= 1000 THEN
        INTO temp_bajos (id, val) VALUES (id, val)
SELECT id, val, sueldo FROM temp_origen;

-- Verificamos
SELECT * FROM temp_altos; -- Debería estar solo Ana
SELECT * FROM temp_bajos; -- Juan y Beto

-- 4. LIMPIEZA
DROP TABLE temp_origen;
DROP TABLE temp_altos;
DROP TABLE temp_bajos;

Distribución Condicional Inteligente

La gerencia quiere separar las ventas históricas usando una sola instrucción INSERT ALL, leyendo desde ventas_historicas:

  • Si el monto es menor o igual a cero, envíalo a ventas_rechazadas. (En la observación pon 'MONTO INVALIDO').
  • Si la zona es 'NORTE' (y monto > 0), envíalo a ventas_norte.
  • Si la zona es 'SUR' (y monto > 0), envíalo a ventas_sur.

Resumen de la Clase 🏆

Checkpoints

SAVEPOINT permite segmentar una transacción grande en etapas seguras, evitando usar ROLLBACK completo ante un error menor.

Subconsultas DML

Permiten dinamismo. Desde crear filtros WHERE inteligentes hasta reemplazar la cláusula VALUES para poblar datos de forma masiva.

INSERT ALL

Una instrucción que procesa una tabla base y distribuye la información hacia múltiples destinos basándose en lógicas condicionales (WHEN).
